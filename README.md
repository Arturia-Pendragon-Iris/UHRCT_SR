# <p align="center">Topology-Preserving Computed Tomography Super-resolution Based on Dual-stream Diffusion Model</p>

This repository is a lightweight, task-oriented reproduction of **Topology-Preserving Computed Tomography Super-Resolution Based on Dual-Stream Diffusion Model**.
The full paper can be found at [link](https://link.springer.com/chapter/10.1007/978-3-031-43999-5_25).


## Method

The reproduced pipeline contains three parts:

1. **Frangi structural target**
   - `conv_filter/filter.py` implements 2D Frangi vesselness for CT slices.
   - It is used to generate reliable vascular/blob-like structural supervision.

2. **Convolutional enhancement operator**
   - `conv_filter/model.py` keeps a lightweight `Filter_UNet`.
   - `conv_filter/train.py` trains this operator to approximate Frangi outputs.
   - This preserves the paper's idea of replacing expensive enhancement operations with a convolutional operator during network optimization.

3. **Dual-stream SR3**
   - `models.py` contains the dual-stream DSSP network.
   - `sr3/diffusion.py` follows SR3/DDPM training and sampling.
   - `sr3/train.py` trains the CT super-resolution model end to end.
   - `sr3/sample.py` generates super-resolved CT slices.

## Data Format

The code expects `.npz` files whose CT array is stored in `arr_0`:

```python
import numpy as np
ct = np.load("case_001.npz")["arr_0"]
```

Supported shapes:

- `H x W`: one 2D CT slice.
- `N x H x W`: a stack of 2D CT slices.

HU values are clipped and normalized from `[-1000, 600]` to `[0, 1]` by default.

## Pretrain the Frangi Approximation Operator

This step trains `Filter_UNet` to approximate true Frangi outputs.

```bash
python -m conv_filter.train \
  --data /path/to/hr_npz \
  --save_dir checkpoints/filter \
  --batch_size 4 \
  --epochs 10 \
  --device cuda
```

The checkpoint is saved as:

```text
checkpoints/filter/filter_unet_last.pth
```

You may skip this step. If `--filter_ckpt` is not provided during SR3 training, the code uses the Torch Frangi implementation directly for structural supervision.

## Train Lightweight Dual-stream SR3

With paired LR/HR `.npz` directories:

```bash
python -m sr3.train \
  --hr_dir /path/to/hr_npz \
  --lr_dir /path/to/lr_npz \
  --filter_ckpt checkpoints/filter/filter_unet_last.pth \
  --save_dir checkpoints/sr3 \
  --crop_size 256 \
  --scale 4 \
  --batch_size 2 \
  --epochs 50 \
  --device cuda
```

With HR data only, LR conditions are generated by bicubic downsampling and upsampling:

```bash
python -m sr3.train \
  --hr_dir /path/to/hr_npz \
  --save_dir checkpoints/sr3 \
  --crop_size 256 \
  --scale 4 \
  --batch_size 2 \
  --epochs 50
```

The SR3 checkpoint is saved as:

```text
checkpoints/sr3/last.pt
```

## Sample

```bash
python -m sr3.sample \
  --input_dir /path/to/lr_or_hr_npz \
  --checkpoint checkpoints/sr3/last.pt \
  --filter_ckpt checkpoints/filter/filter_unet_last.pth \
  --output_dir results/sr3 \
  --sample_steps 100
```

Generated slices are saved as `.npz` files in `results/sr3`.

## Evaluate

```bash
python -m sr3.metrics \
  --pred_dir results/sr3 \
  --gt_dir /path/to/gt_npz
```

The script reports:

- `PSNR`: image-domain fidelity.
- `SMSE`: mean squared error between Frangi structural maps.

## Code Map

```text
models.py              DSSP dual-stream denoising network
utils.py               losses, gradients, Torch Frangi, SMSE
conv_filter/filter.py  NumPy/SciPy Frangi implementation
conv_filter/model.py   Filter_UNet convolutional Frangi approximation
conv_filter/train.py   pretraining script for Filter_UNet
sr3/data.py            CT .npz dataset and synthetic LR generation
sr3/diffusion.py       SR3 diffusion training and sampling
sr3/train.py           end-to-end SR3 training script
sr3/sample.py          inference script
sr3/metrics.py         PSNR and Frangi-SMSE evaluation
```
